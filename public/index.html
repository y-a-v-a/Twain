Twain

A fast, minimal Markdown viewer for macOS. Read-only — no editing, just rendering.

Overview

Twain is a lightweight native macOS app for reading Markdown. It takes a .md file and renders it as a clean, typeset document — headings, lists, tables, links, block quotes, and syntax-highlighted code blocks — without any editing chrome getting in the way. It is a viewer, not an editor: nothing you do in Twain ever writes back to your files.

What sets it apart:

  • Live reload — open documents follow the file on disk, so a window re-renders the moment its source changes (atomic saves included).
  • Native rendering — built on SwiftUI with automatic syntax highlighting in code blocks.
  • Light & dark mode — follows the system appearance, with separate colors for each.
  • Sans-serif or serif — toggle the body font, with your choice (and font size) remembered across launches.
  • Multiple windows — open as many documents as you like, each in its own window.
  • Agent-friendly — drives cleanly from scripts, coding agents, the twain:// URL scheme, and AppleScript.
  • Fully themeable — every color, scale, and spacing value is customizable through one external JSON file.

At roughly a 2 MB release binary, it stays out of the way. This page covers the two things you'll reach for most: theming the look and driving it from scripts.

Keyboard shortcuts

ShortcutAction
Cmd+OOpen file
Cmd+,Open Settings (edit theme)
Cmd++Increase font size
Cmd+-Decrease font size
Cmd+0Reset font size
Cmd+RRefresh file from disk (documents also auto-reload on change)
Cmd+Shift+FToggle serif font

Font size and font-style preferences are saved and restored across app restarts.

Theming

Twain ships with a polished built-in theme, but you can fully customize it through a single JSON file at: 

~/.config/twain/theme.json
Nothing to set up by hand. The file is created automatically on first launch, seeded with the default theme. If it is ever missing or invalid, Twain falls back to the built-in default — so you can't break the app by editing it.

The theme file

The easy way in: open Settings (Cmd+,, under the Twain menu) and click Edit Theme…. That opens theme.json in your default editor, creating it first if needed. Edits are applied live: save the file and every open window re-styles instantly — no restart required.

The theme is grouped into sections. Every section is optional — anything you leave out falls back to its built-in default, which keeps your file forward-compatible as Twain adds options. Colors are written as a pair of hex values, one for light mode and one for dark:

SectionWhat it controls
colorsText, background, link, border, and divider colors (each with a light / dark pair).
headingsfontScales — the size of each heading level H1–H6 relative to body text — and fontWeight.
codeBlockCode-block background, cornerRadius, padding, and fontScale.
blockQuoteborderColor and borderWidth for the quote rule.
paragraphlineSpacingScale (line height) and bottomSpacing between paragraphs.
layoutcontentInset — the padding between content and the window edges.

The default theme

This is the seed file Twain creates for you. Use it as the reference for every available option — copy a section, change a value, save, and watch it apply: 

{
  "colors": {
    "primary":           { "light": "#060606", "dark": "#fbfbfc" },
    "secondary":         { "light": "#6b6e7b", "dark": "#9294a0" },
    "tertiary":          { "light": "#6b6e7b", "dark": "#6d707d" },
    "background":        { "light": "#ffffff", "dark": "#18191d" },
    "secondaryBackground":{ "light": "#f7f7f9", "dark": "#25262a" },
    "link":              { "light": "#2c65cf", "dark": "#4c8ef8" },
    "border":            { "light": "#e4e4e8", "dark": "#42444e" },
    "divider":           { "light": "#d0d0d3", "dark": "#333438" }
  },
  "headings": {
    "fontScales": [2.0, 1.5, 1.25, 1.0, 0.875, 0.85],
    "fontWeight": "semibold"
  },
  "codeBlock": {
    "background": { "light": "#f5f5f8", "dark": "#1f1f24" },
    "cornerRadius": 6,
    "padding": 16,
    "fontScale": 0.85
  },
  "blockQuote": {
    "borderColor": { "light": "#e4e4e8", "dark": "#42444e" },
    "borderWidth": 0.2
  },
  "paragraph": {
    "lineSpacingScale": 0.4,
    "bottomSpacing": 16
  },
  "layout": {
    "contentInset": 32
  }
}

Tips

  • Colors are #RRGGBB hex. Give each color a light and a dark value so it reads well in both appearances.
  • fontScales are multipliers on the body font size — 2.0 means an H1 is twice the body text. The array runs H1 through H6.
  • Want a roomier read? Raise paragraph.lineSpacingScale and layout.contentInset.
  • Only include the sections you want to change — omitted values keep their defaults, so a minimal override file is perfectly valid.
  • Font size and the serif/sans toggle live outside the theme file: adjust them with the keyboard shortcuts above; they persist across restarts.

Automation & scripting

Twain is built to pair with coding agents and scripts. The simplest integration is no integration at all: write a Markdown file and the open window re-renders automatically. For everything else there are two complementary halves — the twain:// URL scheme for driving Twain, and AppleScript for asking it what's open.

URL scheme

The twain:// URL scheme is usable from any process via open:

URLAction
twain://refreshReload every open document from disk
twain://refresh?file=/abs/path.mdReload one document
twain://search?q=textSearch all open documents, jump to the first match
twain://search?q=text&file=/abs/path.mdSearch one open document
twain://open?file=/abs/path.mdOpen a file
twain://open?file=/abs/path.md&search=textOpen and jump to the first match of text
twain://open?file=/abs/path.md&activate=0Open without bringing Twain to the front

File paths must be absolute and query values percent-encoded:

open -g "twain://open?file=/tmp/plan.md&search=Phase%202"

CLI

The installed twain CLI wraps the URL scheme so agents don't have to build URLs by hand: 

twain file.md
twain a.md b.md                 # opens each in its own window
twain -g report.md              # open without stealing focus
twain --find "Install" file.md  # open and jump to the first match
twain --refresh                 # reload every open document from disk
generate-report | twain -       # render stdin

See twain --help for all options.

AppleScript

Where the URL scheme is write-only, AppleScript adds the introspection half — ask Twain what is open and what it shows, via osascript: 

tell application "Twain"
    count documents
    get name of every document
    get path of document 1
    get source text of document 1     -- raw Markdown, follows live reloads
    get rendered text of document 1   -- plain text after Markdown parsing
    refresh document 1                -- re-read from disk
    search document 1 for "Install"   -- open search, jump to first match
    close document 1                  -- closes the window; disk is untouched
end tell

One-liners for scripts and agents:

osascript -e 'tell application "Twain" to get path of every document'
osascript -e 'tell application "Twain" to search document "README.md" for "Theming"'
osascript -e 'tell application "Twain" to close every document'

Documents are addressable by index (registration order) or by file name. The first script that targets Twain triggers a one-time macOS Automation permission prompt for the calling app. The dictionary is defined in Twain.sdef.

Read-only by design. Every automation hook either renders, reloads, searches, or reports — none of them write to your Markdown files. close closes the window; the file on disk is left untouched.
